OWASP Security Checker Application

Examen Cybersecurity Developer, Șpan Lorin, martie 2025.

Broken Access Control

Am implementat mecanisme stricte de control al accesului pentru a preveni accesul neautorizat la resurse.

  • Restricții de acces bazate pe roluri (RBAC) implementate în backend.
  • Protecție împotriva IDOR prin validarea accesului utilizatorului la resursele solicitate.
  • Restricții stricte pe acțiunile privilegiate (modificare, ștergere) în backend.

Cross-Site Scripting (XSS)

Am securizat aplicația împotriva atacurilor XSS prin filtrarea și validarea input-urilor utilizatorilor.

  • Filtrarea input-ului prin sanitizare înainte de a fi afișat în pagină.

Broken Authentication

Am implementat măsuri pentru a asigura autentificarea securizată a utilizatorilor.

  • Politici stricte de parole (minim 7 caractere, obligatoriu cifre, litere mari, mici și caractere speciale).
  • Stocarea sesiunii în backend și utilizarea JWT cu expirare controlată.
  • Limitarea numărului de încercări de login și implementarea rate limiting.

Cryptographic Failures + CSRF

Am eliminat utilizarea algoritmilor de hashing slabi și am implementat protecție CSRF.

  • Utilizarea `bcrypt` pentru hash-ul parolelor, eliminând algoritmi nesiguri precum MD5.
  • Evitat hardcoding-ul de chei criptografice, folosind variabile de mediu pentru gestionarea lor.
  • Protecție CSRF prin utilizarea token-urilor anti-CSRF.

Server-Side Request Forgery (SSRF)

Am limitat accesul cererilor către server pentru a preveni atacurile SSRF.

  • Validarea URL-urilor înainte de a permite accesul serverului la resurse externe.
  • Blocarea accesului către IP-uri private și localhost pentru a preveni atacuri interne.
  • Utilizarea `allowlist` pentru a restricționa solicitările către surse de încredere.

Path Traversal

Am protejat accesul la fișiere pentru a preveni atacurile de tip Path Traversal.

  • Restricționarea accesului la fișierele sensibile prin validarea și normalizarea căii fișierelor.
  • Limitarea accesului utilizatorului doar la directoarele permise.

Remote Code Execution (RCE)

Am securizat serverul pentru a preveni execuția de comenzi arbitrare.

  • Filtrarea și validarea input-ului pentru a preveni injecția de comenzi.
  • Utilizarea de `exec` și `shell_exec` doar în condiții strict controlate.
  • Izolarea aplicației în containere Docker pentru a limita impactul unui atac RCE.
